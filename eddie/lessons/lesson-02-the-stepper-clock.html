The big idea: a stepper motor doesn't rotate. It jumps — one fixed tooth at a time, 1.8° per jump, 200 jumps per revolution. It only looks like rotation when the jumps come fast. Hold that thought: by the end of this lesson it explains how every computer you've ever touched actually works.

⚠ Safety Brief

  • NEVER unplug the motor while the driver is powered. The coils kick back and kill the driver instantly. This is the one mistake on this bench with no fix. Power down first, every time.
  • Power-up order: USB + Pico first, then driver supply. Shutdown in reverse.
  • Motor gets warm at idle — that's normal, it's holding position. Hot enough you can't keep a finger on it = tell the instructor.
  • Clamp the motor. Put a tape flag on the shaft so you can see position.

Driver setup — DIP switches BEFORE power

  • Microstep switches: full step — 200 pulses per revolution. This lesson depends on it.
  • Current switches: match the motor's rated amps (instructor confirms). Too high quietly cooks the motor.

Heads up — the logic is upside down. Because of how we wired it, the Pico fires a step by pulling the pin LOW, not high. The code handles it; you just need to not be surprised. Why it's wired this way is a debrief question.

Chunk 1 — One tooth

# Chunk 1: fire exactly ONE step. One run = one click of the shaft.

from machine import Pin
import time

# Our wiring makes these pins "active low": resting state is HIGH,
# and pulling LOW is what fires. value=1 means "start at rest".
step_pin = Pin(2, Pin.OUT, value=1)
dir_pin  = Pin(3, Pin.OUT, value=1)

def one_step():
    # A step is a quick blink: LOW, tiny pause, back to HIGH.
    # The driver moves the motor one tooth on that blink.
    step_pin.value(0)
    time.sleep_us(10)      # 10 millionths of a second - plenty
    step_pin.value(1)
    time.sleep_us(10)

one_step()
print("One step fired. That was 1.8 degrees. Watch the flag.")
CHECKPOINT — Run it five times. Five distinct clicks of the flag. Each run of the program = one physical event in the world. Sit with that for a second.

Chunk 2 — The visible loop

# Chunk 2: a loop you can WATCH. Twenty steps, one per half second.
# Each pass through the loop = one click of the shaft. The loop
# isn't an idea anymore. It's a ratchet.

from machine import Pin
import time

step_pin = Pin(2, Pin.OUT, value=1)
dir_pin  = Pin(3, Pin.OUT, value=1)

def one_step():
    step_pin.value(0)
    time.sleep_us(10)
    step_pin.value(1)

for count in range(20):          # do the indented lines 20 times
    one_step()
    print("loop pass", count + 1, "- shaft moved one tooth")
    time.sleep(0.5)              # half a second of NOTHING.
                                 # The Pico just stands there.
                                 # That's what sleep means.

print("Done. 20 teeth = 36 degrees.")

Chunk 3 — The illusion of motion

# Chunk 3: the same loop at five different speeds.
# Somewhere in this ladder, clicking becomes motion.
# Nothing about the code changes except the waiting.

from machine import Pin
import time

step_pin = Pin(2, Pin.OUT, value=1)
dir_pin  = Pin(3, Pin.OUT, value=1)

def one_step():
    step_pin.value(0)
    time.sleep_us(10)
    step_pin.value(1)

# Each pair is (steps per second, how many steps to do).
ladder = [(2, 10), (10, 30), (50, 100), (200, 200), (800, 400)]

for speed, steps in ladder:
    print("Now at", speed, "steps per second. Watch AND listen.")
    pause = 1 / speed            # seconds of waiting between steps
    for i in range(steps):
        one_step()
        time.sleep(pause)
    time.sleep(1)                # a breath between rungs

print("Same jumps the whole way. Only the gaps got shorter.")
CHECKPOINT — Name the rung where clicks turned into a hum, and the rung where hum turned into smooth rotation. Write those two numbers down. That boundary is where your eyes stop keeping up with discrete events.
Now the landing: that smooth rotation was never smooth. It was 800 separate jumps per second, faster than your senses sample. Your Pico's processor does the exact same thing — one instruction, then the next, then the next — at 133 million per second. Every "smooth" thing a computer does is the last rung of this ladder. The processor is a stepper with 133 million teeth per second, and time.sleep() is you commanding it to spin the gaps out longer. You just watched a clock cycle with your own eyes.

Chunk 4 — Steps become geometry

# Chunk 4: command ANGLES instead of steps.
# 200 steps = 360 degrees, so degrees-to-steps is just division.

from machine import Pin
import time

step_pin = Pin(2, Pin.OUT, value=1)
dir_pin  = Pin(3, Pin.OUT, value=1)

STEPS_PER_REV = 200              # full-step mode, from the DIP switches

def one_step():
    step_pin.value(0)
    time.sleep_us(10)
    step_pin.value(1)

def turn(degrees, speed):
    # Convert an angle to a whole number of steps.
    steps = int(degrees * STEPS_PER_REV / 360)
    pause = 1 / speed
    print("That angle is", steps, "steps.")
    for i in range(steps):
        one_step()
        time.sleep(pause)

turn(90, 100)     # a quarter turn at 100 steps per second
print("Check the flag: did it land a quarter turn from start?")

Live modifications — all four, verified by the flag:

  1. Command 45°, then 180°, then 720°. Verify each with the flag before the next.
  2. Direction: dir_pin is set once and never touched. Make turn() accept a direction and use it. (LOW is one way, HIGH the other — discover which is which and label it in a comment.)
  3. Make turn() print how long the move should take before it starts. You know steps and pause — the math is yours.
  4. Command 1°. Something's off. What did the flag do, and why? (Look at what int() did to your step count.)
